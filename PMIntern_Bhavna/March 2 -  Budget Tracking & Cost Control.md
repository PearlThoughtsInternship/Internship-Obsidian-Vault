## 3. Budget Tracking Made Simple (15 Minutes)

### 3.1 The Traffic Light System

| Status    | Meaning                                   |
| --------- | ----------------------------------------- |
| 🟢 GREEN  | Under budget (safe)                       |
| 🟡 YELLOW | On budget (monitor closely)               |
| 🔴 RED    | Over budget (action required immediately) |
### Weekly Analysis (Week 7)

- 73% spent vs 65% planned
    
- Status: 🔴 RED
    

**Reason:** SMS costs doubled expectation  
**Fix:** Reduce SMS testing volume

### 3.2 Cost Categories to Watch

#### 🔴 High Risk Areas

- Cloud costs (auto scaling increases bill)
    
- Third-party APIs (Twilio, Razorpay – usage based)
    
- Overtime hours
    
- Scope creep (new features)
    

#### 🟢 Low Risk Areas

- Jira licenses
    
- Basic design tools

## 4. Cost Alerts & Action Triggers (10 Minutes)

### 4.1 When to Panic

| Alert Level               | Condition                             |
| ------------------------- | ------------------------------------- |
| Alert Level 1 (Yellow)    | Any category >110% planned            |
| Alert Level 2 (Red)       | Total spend >85% before 80% work done |
| Alert Level 3 (Emergency) | Any week >15% budget burn             |
|                           |                                       |
|                           |                                       |
### Schedula Week 7 Alerts

- Twilio: 180% of planned → Alert Level 1
    
- Total spend: 73% vs 65% work done → Alert Level 2

### 4.2 Three Ways to Save Money

1. **DEFER** – Move feature to Phase 2
    
2. **REDUCE** – Limit usage (e.g., SMS cap 5000/month)
    
3. **NEGOTIATE** – Volume discounts / startup pricing
