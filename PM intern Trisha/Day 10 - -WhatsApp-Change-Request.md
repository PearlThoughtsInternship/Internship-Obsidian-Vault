1. Change Request Overview

Requested By: Salon Owner (Client)
Client Priority Level: High

Request Description

The client has proposed adding a WhatsApp-based booking system that allows customers to schedule appointments directly through WhatsApp. The feature should allow users to:

Book available time slots

Receive booking confirmations

Get automated reminders before appointments

Business Justification

Currently, most salon customers come through social media inquiries and walk-in visits. Since WhatsApp is widely used for communication, integrating booking functionality into WhatsApp could:

Simplify appointment scheduling

Reduce response time

Improve customer engagement

Increase operational efficiency

2. Scope Impact Analysis

This request introduces a new feature that was not included in the original MVP plan. It involves technical changes and integration with an external platform.

Area	Current Plan	After Change	Impact
Story Points	18 SP	23 SP	+5 SP
Features	Standard booking only	WhatsApp booking added	New feature module
UI Changes	Basic interface	Additional booking & confirmation flow	Moderate effort
External Integration	None	WhatsApp Business API required	New dependency

The scope increases by approximately 28%, which exceeds the recommended safe change limit within a sprint.

6. Available Options
Option A – Implement in Current Sprint

Introduce WhatsApp integration immediately.

Advantages:

Feature available at launch

Better customer convenience

Competitive edge

Disadvantages:

Launch may be delayed by 1–2 weeks

Increased development cost

Higher pressure on the team

Option B – Postpone to Phase 2 (Recommended)

Release MVP as planned and add WhatsApp booking in the next release cycle.

Advantages:

Launch timeline remains unchanged

Budget remains under control

Lower execution risk

Feature can be promoted as “Coming Soon”

Disadvantages:

Feature unavailable during initial launch

Client waits a few weeks

Option C – Scope Adjustment

Add WhatsApp booking but remove the Monthly Reports feature from MVP.

Advantages:

Timeline remains stable

Budget impact minimized

WhatsApp feature available immediately

Disadvantages:

Reporting functionality temporarily unavailable

Business analytics insights delayed

7. Final Recommendation

Recommended Option: Option B – Move to Phase 2

Justification:

Meeting the planned launch date is important due to the upcoming wedding season demand.

External API approvals may cause unexpected delays.

Adding the feature now increases both cost and risk.

Focusing on a stable MVP ensures better product quality.

Proposed Phase 2 Plan

Planning: Early March

Development & Testing: 1 week

Release Target: Mid-March

8. Communication Strategy
